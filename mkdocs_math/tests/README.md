# mkdocs-math Plugin Tests

Integration tests for the mkdocs-math plugin.

## Testing Philosophy

We validate the plugin by building a **single realistic test project** and running **multiple validation layers** against the same build artifacts.

This approach:
- **Reflects real usage**: Build once (like a user would), then validate
- **Efficient**: One build, many tests
- **Clear failure modes**: Each layer detects different types of issues
- **Scalable**: As we add features, we add more validations against the same project

## Three-Layer Validation Strategy

Each feature (like preamble injection or environment rendering) is tested in three stages:

### Layer 1: Build Validation
Does the project build successfully with the plugin?

**Example tests**:
- Build completes without errors
- Build output directory is created
- Expected pages exist

**Detects**: Plugin crashes, configuration errors, missing dependencies

### Layer 2: HTML Inspection
Is the feature correctly injected into the HTML output?

**Example tests** (for preamble):
- LaTeX commands are present in HTML
- Commands use correct delimiters (`$$` not `<code>`)
- HTML structure is valid

**Example tests** (for environments):
- Environment markers are present
- Math notation in labels is preserved
- Consecutive environments are properly separated

**Detects**: Injection logic bugs, format issues, markdown processing problems

### Layer 3: Browser Rendering
Does the feature actually work in a real browser with JavaScript?

**Example tests** (for preamble):
- MathJax renders math elements
- Preamble commands are processed (not left as raw LaTeX)

**Example tests** (for environments):
- Environments render without errors
- Math notation is rendered by MathJax

**Detects**: MathJax configuration issues, rendering problems, subtle formatting bugs

## Mathematical Environments Specification

Mathematical environments are defined using bold text followed by a period:

### Syntax Rules

- **Format**: `**EnvironmentName.**` or `**EnvironmentName (Label).**`
- **Period required**: Must end with `.**` (e.g., `**Theorem.**` not `**Theorem**`)
- **Termination**: End environments with two blank lines (three newlines in a row)
- **Environment types**: Any capitalized name works (e.g., Theorem, Proposition, Lemma, Corollary, Definition, Example, Remark, Proof)

### Basic Example

```markdown
**Theorem (Pythagorean).**
For a right triangle with sides $a$, $b$ and hypotenuse $c$:
$$
a^2 + b^2 = c^2
$$


**Proof.**
By the Pythagorean theorem...


Regular text continues here.
```

### Labels and Cross-References

Include a label in parentheses to create an anchor for linking:

```markdown
**Proposition (Weak Flatness).**
For $a \in C(X)$...


**Corollary (Direct Consequence).**
As a direct consequence...


See [Proposition on Weak Flatness](#proposition-weak-flatness).
```

Labels are automatically converted to IDs:
- Special characters removed
- Spaces become hyphens
- All lowercase
- Example: `**Proposition (Weak Flatness).**` → `id="proposition-weak-flatness"`

### Math in Environments

Math notation in labels and content is fully supported:

```markdown
**Definition** ($\mathbb{R}^n$).
A Euclidean space is the vector space $\mathbb{R}^n$ equipped with the standard inner product.


**Theorem** (Compact Open Topology).
Let $X$ and $Y$ be topological spaces. The compact-open topology on $C(X, Y)$ is generated by subbasic sets of the form:
$$\{f \in C(X, Y) : f(K) \subseteq U\}$$
where $K \subseteq X$ is compact and $U \subseteq Y$ is open.


**Proof**.
The existence follows from general topology...
```

## Future Direction

As the plugin grows:

1. **More validations against one project**: Add more tests for new features
2. **Separate test projects for different scenarios**: Create additional projects for edge cases
3. **Permutation testing**: Test combinations of features
4. **Performance benchmarks**: Validate plugin performance

The key principle: **Build realistic projects once, validate thoroughly**.

## Running Tests

```bash
# All tests
pytest mkdocs_math/tests/ -v

# Preamble tests only
pytest mkdocs_math/tests/test_preamble_injection.py -v
pytest mkdocs_math/tests/test_preamble_rendering.py -v

# Environment tests only
pytest mkdocs_math/tests/test_environments.py -v

# Via Makefile
make mkdocs-math-test
```
